## MSA과정 운영에 필요한 추가 기능

### 1. 관찰일지 관리 (신규)

> 강사가 주 1회 학생별 대인관계/문제해결 능력을 평가하는 기능

**필요 Entity**
```
ObservationLog (관찰일지)
├── id, course, team, observationDate, round (차수)
├── observer (강사)
├── generalComment (종합 소견)
└── 1:M → ObservationRecord (학생별 기록)
           ├── student
           ├── interpersonalScore (대인관계 1~5)
           ├── problemSolvingScore (문제해결 1~5)
           ├── behaviorNote (행동 근거)
           └── evidenceChecklist (근거자료 체크)
```

**필요 화면**
- 관찰일지 입력 폼 (팀 선택 → 학생별 점수 + 코멘트)
- 관찰일지 목록/상세 조회
- 관찰일지 PDF 출력

**필요 API**
```
POST /api/observation/save              # 관찰일지 등록
GET  /api/observation/course/{id}/list  # 과정별 관찰일지 목록
GET  /api/observation/{id}              # 관찰일지 상세
GET  /api/observation/{id}/pdf          # PDF 출력
```

### 2. 프로젝트 평가표 관리 (신규)

> 프로젝트 결과물에 대한 6개 영역 100점 만점 종합 평가

**현재 상태**: Paper에 PROJECT 타입은 있으나, 6개 영역(계획/내용/수준/기능/태도/발표) 세부 채점 불가

**필요 Entity**
```
ProjectEvaluation (프로젝트 평가표)
├── id, course, team, evaluationType (중간/최종)
├── evaluationDate, evaluator (강사)
├── totalScore (합계)
├── grade (A~F)
├── 1:M → ProjectEvalScore (영역별 점수)
│          ├── category (계획/내용/수준/기능수준/태도/구두발표)
│          ├── item (세부 항목명)
│          ├── maxScore (배점)
│          ├── score (득점)
│          └── evidence (채점 근거)
└── summary (총평 6개 항목: 완성도, 기술성취, 협업, PBL, 개선, 종합)
```

**필요 화면**
- 프로젝트 평가표 입력 폼 (17개 항목 점수 + 총평)
- 평가표 목록/상세
- 평가표 PDF 출력

**필요 API**
```
POST /api/project-eval/save             # 평가표 등록
GET  /api/project-eval/course/{id}/list # 과정별 평가표 목록
GET  /api/project-eval/{id}             # 평가표 상세
GET  /api/project-eval/{id}/pdf         # PDF 출력
```

### 3. 멘토링일지 관리 (신규)

> 참여기업 멘토링 내용을 기록하고 서류를 자동 생성

**필요 Entity**
```
MentoringLog (멘토링일지)
├── id, course, team, round (차수)
├── mentoringDate, mentoringType (대면/비대면)
├── mentorName, mentorCompany, mentorCareer
├── attendees (참석 인원)
├── 1:M → MentoringQA (질문/답변)
│          ├── question, answer
├── feedbackGood (잘하는 점)
├── feedbackImprove (개선점)
├── feedbackResource (추천 자료)
├── 1:M → MentoringAction (액션아이템)
│          ├── content, assignee, deadline, isDone
├── mentorSign, instructorSign
```

**필요 화면**
- 멘토링일지 입력 폼
- 멘토링일지 목록/상세
- 멘토링일지 PDF 출력

**필요 API**
```
POST /api/mentoring/save              # 멘토링일지 등록
GET  /api/mentoring/course/{id}/list  # 과정별 멘토링 목록
GET  /api/mentoring/{id}              # 멘토링일지 상세
GET  /api/mentoring/{id}/pdf          # PDF 출력
```

### 4. 팀 관리 (신규)

> 현재 ncs-ssr에는 팀 개념이 없음. 프로젝트 운영을 위해 팀 관리 필요

**필요 Entity**
```
Team (팀)
├── id, course, teamName
├── 1:M → TeamMember
│          ├── student
│          ├── role (팀장/백엔드리드/프론트리드/DB담당/QA)
├── notionUrl (노션 링크)
├── githubUrl (GitHub 레포 링크)
├── projectTitle (프로젝트명)
```

**필요 화면**
- 팀 구성 폼 (학생 배정, 역할 지정)
- 팀 목록/상세 (노션/GitHub 링크 포함)

**필요 API**
```
POST /api/team/save                   # 팀 등록
GET  /api/team/course/{id}/list       # 과정별 팀 목록
GET  /api/team/{id}                   # 팀 상세
PUT  /api/team/{id}                   # 팀 정보 수정
```

### 5. 제출 서류 대시보드 (신규) - 단일 진입점

> **15종 국가 제출 서류를 한 화면에서 전부 확인하는 단일 진입점**
> 강사 서류(PDF), 학생 서류(노션 링크), 소스코드(GitHub 링크)를 하나의 대시보드에서 접근

**핵심 개념**: 팀 등록 시 노션 URL + GitHub URL을 입력받아, 대시보드에서 링크로 연결

**필요 Entity 확장** (Team에 추가)
```
Team (팀) - 기존 필드에 추가
├── notionUrl        (팀 노션 워크스페이스 URL)
├── githubUrl        (팀 GitHub 레포 URL)
└── 1:M → DocumentStatus (서류별 상태)
           ├── documentType (15종 서류 구분)
           ├── status (미작성/작성중/완료)
           └── externalUrl (노션 개별 페이지 URL, 선택)
```

**필요 화면**

1. **과정별 팀 목록 (메인 대시보드)**
   - 팀별 프로젝트명, 노션/GitHub 링크, 서류 진행률 (n/15)
   - 한눈에 전체 현황 파악

2. **팀별 서류 상세 화면**
   - 15종 서류를 3개 그룹으로 표시
     - 강사 서류 (홈페이지 내부) → [PDF 보기] 버튼
     - 학생 서류 (노션) → [노션 열기] 버튼 (링크 이동)
     - 소스코드 (GitHub) → [GitHub 열기] 버튼 (링크 이동)
   - 서류별 상태 표시 (미작성/작성중/완료)
   - [PDF 일괄 다운로드] 버튼 → 강사 서류 전체 ZIP

3. **팀 등록/수정 폼**
   - 노션 URL 입력 필드
   - GitHub URL 입력 필드

**필요 API**
```
GET  /api/dashboard/course/{id}                # 과정별 팀 목록 + 서류 현황
GET  /api/dashboard/team/{id}                  # 팀별 15종 서류 상세
PUT  /api/dashboard/team/{id}/document-status  # 서류 상태 업데이트
GET  /api/dashboard/team/{id}/download-all     # 강사 서류 PDF 일괄 다운로드
```

### 6. 기존 기능 개선

#### 6-1. PBL 평가 강화
- 현재: Paper에 pblTitle, pblScenario 등 기본 필드만 존재
- 필요: 프로젝트 평가표(6개 영역)와 연동, 팀 단위 평가 지원

#### 6-2. 동료평가 온라인화 (선택)
- 현재: 노션에서 수기 작성
- 가능: 평가사이트에서 팀원 간 상호 평가 입력 → 집계/PDF 자동 생성

#### 6-3. 보고서 확장
- No.6: 관찰일지 종합 보고서 (주차별 전체 학생 대인관계/문제해결 추이)
- No.7: 프로젝트 평가 종합 보고서 (팀별 비교)
- No.8: 멘토링 종합 보고서 (4차 멘토링 요약)

## 기능별 우선순위

| 우선순위 | 기능 | 이유 |
|---------|------|------|
| **필수** | 관찰일지 관리 | 매주 작성, 국가 제출 필수 |
| **필수** | 프로젝트 평가표 | 중간/최종 평가, 국가 제출 필수 |
| **필수** | 멘토링일지 관리 | 4회 작성, 국가 제출 필수 |
| **필수** | 팀 관리 | 관찰일지/평가표가 팀 단위로 동작 |
| **필수** | 제출 서류 대시보드 (단일 진입점) | 15종 서류 한 화면 확인, 국가 심사 대응 |
| **선택** | 동료평가 온라인화 | 노션으로 대체 가능 |
| **선택** | 보고서 확장 (No.6~8) | 기존 5종으로도 제출 가능 |
