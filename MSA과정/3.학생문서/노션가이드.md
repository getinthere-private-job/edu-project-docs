# 프로젝트 노션 가이드

## 노션 구조

팀 노션 워크스페이스에 아래 페이지를 만들어주세요.

```
프로젝트1/
├── PBL 프로젝트 계획서  (팀)
├── 회의록             (팀)
├── 팀블로그관리        (팀)
├── 코드리뷰           (팀, 스텝 완료시)
├── 트러블슈팅         (팀)
├── 프로젝트회고        (팀)
├── 최종산출물          (팀 - PPT, 시연영상, 발표영상)
└── 하브루타노트        (개인별 각자)
```

> 최소 요구사항은 강사가 PBL 프로젝트 계획서 템플릿으로 제공합니다.

---

## 문서별 작성 가이드

### 1. PBL 프로젝트 계획서 (팀)

프로젝트 초기에 작성하고, 스프린트마다 업데이트합니다.
강사가 제공한 최소 요구사항 템플릿을 기반으로 작성하세요.

**초기 작성 항목:**
- 프로젝트 개요 (한 줄 소개, 목적, 대상 사용자)
- 문제 정의 (어떤 문제를 해결하는지)
- 기능 목록 (MVP 필수 기능 + 단계별 추가 기능)
- 기술 스택 선정 이유
- DB 설계 초안 (ERD)
- API 설계 초안 (REST API 엔드포인트 목록)
- 일정표 (WBS, 4주 마일스톤)
- 리스크 분석
- Git Flow (브랜치 전략, 커밋 컨벤션)

**스프린트별 업데이트 (하단에 추가):**
- 스프린트 번호, 기간, 목표
- 백로그: 기능, 담당자, 예상시간, 우선순위, 상태
- MVP 진행률 체크
- KPT 회고 (Keep / Problem / Try)

### 2. 회의록 (팀)

- 일자, 참석자, 안건
- 회의 내용 정리
- 회의 사진 첨부 (근거자료용)
- 팀원별 진행 상황 (일일 스탠드업)

### 3. 팀블로그관리 (팀)

| 팀원 | 블로그 URL | 작성 글 수 |
|------|-----------|-----------|
| | | |

- 주간 발표 기록 (발표자, 주제, 날짜)
- 블로그 글 노션에 복제 (근거자료용)

### 4. 코드리뷰 (팀, 스텝 완료시)

특정 스텝(회원가입, 핵심CRUD, Redis 등) 완료 후 master에 커밋하면 강사가 리뷰합니다.
강사 리뷰 결과를 이 문서에 기록하세요.

- 스텝명 (예: 회원관리 완료)
- master 커밋 링크
- 강사 리뷰 내용 요약
- 개선 사항 및 반영 여부

### 5. 트러블슈팅 (팀)

| 항목 | 내용 |
|------|------|
| 증상 | 어떤 에러/문제가 발생했는지 |
| 원인 | 왜 발생했는지 |
| 해결 | 어떻게 해결했는지 |
| 배운 점 | 이 경험에서 무엇을 배웠는지 |

### 6. 프로젝트회고 (팀)

- 프로젝트 요약 (한 줄)
- KPT 회고 (Keep / Problem / Try)
- 팀원별 한 줄 회고

### 7. 최종산출물 (팀)

프로젝트 완료 시 아래 3가지를 준비하여 노션에 첨부/링크합니다.

| 산출물 | 설명 | 형식 |
|--------|------|------|
| **PPT** | 최종 발표용 프레젠테이션 | PDF 또는 PPT 파일 첨부 |
| **시연영상** | 프로젝트 주요 기능 시연 녹화 | 유튜브/구글드라이브 링크 |
| **발표영상** | 최종 발표 녹화 영상 | 유튜브/구글드라이브 링크 |

### 8. 하브루타노트 (개인별)

주 2~3회 이상 작성. 날짜별로 기록.

| 항목 | 내용 |
|------|------|
| What | 오늘 무엇을 했는지 |
| Why | 왜 그것을 했는지 |
| Problem | 어떤 문제가 있었는지 |
| Solution | 어떻게 해결했는지 |
| 회고 | 느낀 점 |

## 제출 방법

### 홈페이지에 노션 URL 등록 (필수)

팀 등록 시 **노션 워크스페이스 URL**을 홈페이지에 입력하세요.
등록하면 강사와 국가 심사관이 **홈페이지 대시보드**에서 노션 링크를 클릭하여 바로 확인할 수 있습니다.

> 작성은 노션에서, 확인은 홈페이지 대시보드 한 곳에서!

### 등록할 URL

| 항목 | 등록 위치 | 예시 |
|------|----------|------|
| 팀 노션 URL | 홈페이지 팀 등록 | https://notion.so/team1-project |
| GitHub URL | 홈페이지 팀 등록 | https://github.com/team1/project |

### 추가 제출

- 노션 → PDF 내보내기 또는 캡처 (국가 제출 시 필요할 수 있음)
- GitHub URL은 홈페이지 팀 등록 시 이미 입력됨
