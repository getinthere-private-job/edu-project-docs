# 평가 근거자료 가이드

> 본 문서는 프로젝트 평가 시 확인해야 할 근거자료와 확인 방법을 정리한 가이드입니다.
> 상세 평가 기준은 `1.운영문서/프로젝트운영.md` Section 4를 참조하세요.

---

## 평가 구조

| 구분 | 배점 | 설명 |
|------|------|------|
| A. 팀 평가 | 50점 | 팀 전체 결과물 + 협업 프로세스 (팀원 동일 점수) |
| B. 개인 평가 | 50점 | 태도(20) + 지식(20) + 성장(10) (개인별 채점) |
| **합계** | **100점** | |

---

## 근거자료 확인 경로

| 근거자료 | 확인 위치 | 평가 영역 | 확인 방법 |
|---------|----------|----------|----------|
| 회의록 (참석/불참) | 노션 | 태도 | 대시보드 → [노션 열기] → 회의록 페이지 |
| 회의 사진 | 노션 회의록 첨부 | 태도 | 회의록 내 사진 확인 |
| 스탠드업 발언 기록 | 노션 회의록 | 태도 | 팀원별 진행 상황 섹션 |
| GitHub 개인 커밋 | GitHub Contributors | 태도, 지식 | 대시보드 → [GitHub 열기] → Insights → Contributors |
| GitHub PR/코드리뷰 | GitHub Pull Requests | 태도, 지식 | 대시보드 → [GitHub 열기] → Pull Requests |
| 스프린트 백로그 | 노션 PBL 프로젝트 계획서 | 태도 | 스프린트 업데이트 섹션의 백로그 상태 |
| 하브루타노트 | 노션 | 지식, 성장 | 대시보드 → [노션 열기] → 개인별 하브루타노트 |
| 블로그 작성 내역 | 노션 팀블로그 | 성장 | 블로그 URL + 노션 복제본 |
| 트러블슈팅 문서 | 노션 | 지식, 성장 | 증상→원인→해결→배운점 기록 확인 |
| 강사 코드리뷰 결과 | 노션 코드리뷰 기록 | 지식 | 스텝별 리뷰 내용 + 개선 반영 여부 |
| 관찰일지 누적 기록 | 홈페이지 | 태도, 지식, 성장 | 대시보드 → [보기] → 주차별 점수 확인 |

---

## 평가 프로세스

```
평가 전
├─ 근거자료 체크리스트 확인 (위 표 기준)
├─ 대시보드에서 노션/GitHub 링크 접속하여 근거 확인
│
평가 시
├─ A. 팀 평가 (50점): 계획 → 구현 → 산출물 → 발표
├─ B. 개인 평가 (50점): 태도 → 지식 → 성장
├─ 개인별 채점표 작성
├─ 총평 작성 (팀 + 개인별)
│
평가 후
├─ [홈페이지] 프로젝트 평가표 입력 (출력 불필요)
└─ [홈페이지] 포트폴리오 평가 채점 (PDF 출력 필요)
```
