# PBL 프로젝트 최소 요구사항 (강사 제공 템플릿)

> 본 문서는 강사가 학생에게 제공하는 최소 요구사항 템플릿입니다.
> 학생은 이 템플릿을 기반으로 PBL 프로젝트 계획서를 작성합니다.
> 상세 내용은 `1.운영문서/프로젝트운영.md` Section 2-1을 참조하세요.

## 프로젝트 최소 요구사항

| 번호 | 최소 요구사항 | 기준 |
|-----|-------------|------|
| 1 | REST API | 최소 3개 이상 (Ajax 사용) |
| 2 | DB 테이블 | 4개 이상 (JPA 연관관계 매핑 포함) |
| 3 | 인증/인가 | 회원가입, 로그인, 역할 구분 (최소 2개 역할) |
| 4 | Redis 세션 관리 | 세션 저장소를 Redis로 구성 |
| 5 | Mustache SSR | Server Side Rendering 화면 구성 |
| 6 | 예외처리 및 유효성 검증 | 글로벌 예외 핸들러 + 서버 사이드 검증 |
| 7 | 페이징/정렬 | 목록 조회 시 페이징 기능 |
| 8 | Git Flow | 브랜치 전략 수립, 스텝 완료 시 master 커밋 |
| 9 | README | 아키텍처 다이어그램 포함 |
