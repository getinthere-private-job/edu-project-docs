# 문서 가이드

## 제출 서류 2대 분류

| 분류 | 내용 | PDF 출력 | 확인 방법 |
|------|------|---------|----------|
| **PBL 프로젝트 문서** | 강사 입력 + 학생 문서 + 코드 | 불필요 | 홈페이지/노션/GitHub에 존재, 대시보드 링크로 확인 |
| **시험 문서** | 개인별 이론평가 + 실습평가 | **필요** | 홈페이지에서 시험 → PDF 자동생성 → 다운로드 |

## 단일 진입점: 홈페이지 대시보드

> **작성은 각자 편한 곳에서, 확인은 홈페이지 한 곳에서!**

| 도구 | 용도 | 누가 | 대시보드 연결 |
|------|------|------|-------------|
| **홈페이지** | 시험(PDF) + PBL 강사 입력(보기만) + **대시보드** | 강사 | 직접 보기/PDF 다운로드 |
| **노션** | PBL 프로젝트 학생 문서 8종 작성 | 학생 | 노션 링크로 연결 |
| **GitHub** | 코드 + 스텝 완료시 master 커밋 → 강사 코드리뷰 | 학생 | GitHub 링크로 연결 |

팀 등록 시 **노션 URL**과 **GitHub URL**을 홈페이지에 입력하면,
대시보드에서 PBL 문서 + 시험 문서를 한 화면으로 전부 확인할 수 있습니다.

## 1. 시험/채점 관리 (PDF 출력 필요)

### 시험 종류

| 구분 | 평가 내용 | 도구 | 시기 | PDF |
|------|----------|------|------|-----|
| 개인 이론평가 | 객관식 시험 → 자동 채점 | 홈페이지 | 중간/최종 | 채점지 PDF |
| 개인 실습평가 | 포트폴리오 평가 → 강사 채점 | 홈페이지 | 중간/최종 | 채점지 PDF |
| 보고서 5종 | 평가계획/문항분석/문항지/성적분석/통계 | 홈페이지 | 자동생성 | PDF |

### 시험 프로세스

```
[홈페이지에서 진행]

시험 준비
├─ 시험지 등록 (객관식/포트폴리오)
├─ 문제 등록
│
중간/최종 시험
├─ 학생: 객관식 시험 응시 → 자동 채점
├─ 강사: 포트폴리오 평가 채점
├─ 자동: 보고서 5종 생성
│
결과
└─ 채점지 + 보고서 PDF 다운로드 (출력 필요)
```

## 2. PBL 프로젝트 문서 관리 (출력 불필요)

> PBL 프로젝트 문서는 노션/홈페이지에 존재만 하면 됨. 출력할 필요 없음.

### 강사 입력 (홈페이지에서 보기만)

| 문서 | 시기 | 입력 도구 | 비고 |
|------|------|----------|------|
| PBL 프로젝트 계획서 (템플릿) | 프로젝트 시작 전 | 홈페이지 | 최소 요구사항 템플릿 제공 |
| 관찰일지 | 주 1회 | 홈페이지 | 학생별 점수 입력 → 보기만 |
| 프로젝트 평가표 | 중간/최종 | 홈페이지 | 팀50점+개인50점=100점 → 보기만 |
| 멘토링일지 | 멘토링 후 (4회) | 홈페이지 | Q&A + 피드백 → 보기만 |

### 학생 작성 문서 8종 (노션에서 작성, 링크로 확인)

> 최소 요구사항은 강사가 PBL 프로젝트 계획서 템플릿으로 제공합니다.

| 문서 | 작성 단위 | 작성 시기 | 제출 근거 |
|------|----------|----------|----------|
| PBL 프로젝트 계획서 | 팀 | 프로젝트 초기 + 스프린트마다 업데이트 | 프로젝트 개요, MVP, 기술스택, WBS, KPT |
| 회의록 | 팀 | 매일 | 안건, 사진, 스탠드업 |
| 팀블로그관리 | 팀 | 수시 | 블로그 현황, 발표기록, 노션 복제 |
| 코드리뷰 | 팀 | 스텝 완료시 | master 커밋 강사 리뷰 결과 기록 |
| 트러블슈팅 | 팀 | 에러 발생시 | 증상→원인→해결→배운점 |
| 프로젝트회고 | 팀 | 종료 시 | KPT, 팀원별 회고 |
| 최종산출물 | 팀 | 종료 시 | PPT, 시연영상, 발표영상 |
| 하브루타노트 | 개인 | 주 2~3회 | What/Why/Problem/Solution/회고 |

### 평가사이트에 추가할 입력 폼 (PBL용, 출력 불필요)

**관찰일지 폼** (주 1회)

| 입력 항목 | 타입 |
|----------|------|
| 관찰 차수 | 선택 (1차~4차) |
| 팀 선택 | 드롭다운 |
| 학생별 대인관계 능력 | 1~5점 |
| 학생별 문제해결 능력 | 1~5점 |
| 학생별 관찰 내용 | 텍스트 |
| 근거자료 체크 | 체크박스 (GitHub, 회의록, 하브루타, 블로그, 스탠드업) |
| 종합 소견 | 텍스트 |

**평가표 폼** (중간/최종)

| 입력 항목 | 타입 |
|----------|------|
| 평가 유형 | 중간/최종 선택 |
| 팀 선택 | 드롭다운 |
| **[A. 팀 평가 50점]** | |
| 계획 (10점) | 2개 항목 점수 (계획서 완성도 5점, 설계 적합성 5점) |
| 구현 (20점) | 3개 항목 점수 (MVP 7점, 기술스택 7점, 코드품질 6점) |
| 산출물 (10점) | 2개 항목 점수 (노션 문서 5점, GitHub 관리 5점) |
| 발표 (10점) | 2개 항목 점수 (발표+시연 6점, 질의응답 4점) |
| **[B. 개인 평가 50점 × 학생수]** | |
| 학생별 태도 (20점) | 4개 항목 점수 (출석 6, 스탠드업 4, 코드리뷰 5, 일정 5) |
| 학생별 지식 (20점) | 3개 항목 점수 (커밋 6, 하브루타 6, 구현수준 8) |
| 학생별 성장 (10점) | 3개 항목 점수 (블로그 4, 트러블슈팅 3, 자기주도 3) |
| 팀 총평 | 텍스트 (완성도, 기술적 성취, PBL 학습 성과) |
| 학생별 총평 | 텍스트 (강점, 개선 사항) |

**멘토링일지 폼** (4회)

| 입력 항목 | 타입 |
|----------|------|
| 멘토링 차수 | 선택 (1차~4차) |
| 멘토 정보 | 이름, 소속, 유형(대면/비대면) |
| 대상 팀 | 드롭다운 |
| 질문/답변 | 반복 입력 (번호, 질문, 답변) |
| 멘토 피드백 | 잘한점, 개선점, 추천자료 |
| 액션 아이템 | 반복 입력 (내용, 담당자, 기한) |

## 3. 프로세스 요약

### 강사 프로세스

```
프로젝트 시작 전
│
├─ PBL 계획서 작성 (이미 완료)
├─ [홈페이지] 팀 등록 (노션 URL + GitHub URL 입력)
├─ [홈페이지] 시험지 등록 (객관식/포트폴리오)
│
프로젝트 진행 중
│
├─ 매일:  스탠드업 참관, 질의응답 대응
│         [대시보드 → 노션/GitHub 링크] 근거자료 확인
├─ 스텝 완료시: GitHub master 커밋 코드리뷰
├─ 주 1회: 관찰일지 입력 (홈페이지, 출력 불필요)
├─ 멘토링 후: 멘토링일지 입력 (홈페이지, 출력 불필요)
│
├─ 중간 발표
│   ├─ 평가표 입력 (홈페이지, 출력 불필요)
│   ├─ 객관식 시험 진행 → 채점지 PDF 자동생성
│   └─ 포트폴리오 평가 → 채점지 PDF 자동생성
│
└─ 최종 발표
    ├─ 평가표 입력, 시험 진행 → PDF 자동생성
    └─ [대시보드] PBL 문서=링크 확인 / 시험 PDF=다운로드
```

### 학생 프로세스

```
프로젝트 시작
│
├─ 1주차: PBL 프로젝트 계획서 작성 (노션)
│
├─ 매일:  회의록 + 사진 (노션) / 하브루타노트 (노션)
│         코드 작업 (GitHub, feature 브랜치)
│
├─ 스텝 완료시: master 커밋 → 강사 리뷰 → 리뷰 결과 기록 (노션)
├─ 에러시: 트러블슈팅 기록 (노션)
├─ 스프린트마다: PBL 프로젝트 계획서 업데이트 + KPT 회고 (노션)
├─ 수시: 블로그 작성 → 노션 복제
│
├─ 중간/최종
│   └─ [홈페이지] 시험 응시 (객관식/포트폴리오)
│
└─ 종료: 프로젝트회고 (노션) + 최종산출물 준비 (PPT/시연영상/발표영상) + 모든 문서 정리
```

### 분류 2. 시험 문서 (PDF 출력 필요)

| 구분 | 서류 | 생성 방식 | PDF |
|------|------|----------|-----|
| 이론평가 | 객관식 채점지 (중간/최종) | 시험 → 자동 채점 | [PDF 다운로드] |
| 실습평가 | 포트폴리오 채점지 (중간/최종) | 강사 채점 | [PDF 다운로드] |
| 보고서 | No.1~5 (평가계획/문항분석/문항지/성적분석/통계) | 자동생성 | [PDF 다운로드] |
| 기타 | 총평 | 자동생성 | [PDF 다운로드] |
